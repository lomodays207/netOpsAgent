# 如何检查服务监听状态

检查服务监听状态是网络故障排查中的关键步骤，用于确认服务是否正常启动以及是否在预期的端口上监听。

## 常用命令

### 1. netstat 命令

`netstat` 是最常用的网络统计工具。

- **Windows**:
  ```cmd
  netstat -ano | findstr <端口号>
  ```
  - `-a`: 显示所有连接和监听端口
  - `-n`: 以数字形式显示地址和端口号
  - `-o`: 显示拥有每个连接的进程 ID (PID)

- **Linux**:
  ```bash
  netstat -tuln | grep <端口号>
  ```
  - `-t`: TCP 协议
  - `-u`: UDP 协议
  - `-l`: 仅显示监听状态的 socket
  - `-n`: 以数字形式显示地址和端口号

### 2. ss 命令 (Linux)

`ss` 命令比 netstat 更快，是 Linux 上推荐的替代工具。

```bash
ss -tuln | grep <端口号>
```

### 3. lsof 命令 (Linux)

`lsof` (List Open Files) 也可以查看端口监听情况。

```bash
lsof -i :<端口号>
```

### 4. PowerShell (Windows)

```powershell
Get-NetTCPConnection -LocalPort <端口号>
```

## 结果分析

- **LISTENING**: 表示服务正在监听，如果看到此状态且 IP 是 `0.0.0.0` 或 `::`，说明服务对外开放。如果是 `127.0.0.1`，则只能本地访问。
- **ESTABLISHED**: 表示已建立连接。
- **TIME_WAIT**: 表示连接已关闭，正在等待结束。
- **无输出**: 表示该端口没有被任何进程监听，服务可能未启动或使用了错误的端口。

## 常见问题

1. **服务未启动**: 检查服务状态（如 systemctl status nginx）。
2. **防火墙拦截**: 即使端口在监听，防火墙也可能阻止外部访问。
3. **绑定地址错误**: 服务可能只绑定了 127.0.0.1，导致外部无法访问。确保服务配置文件中绑定地址为 0.0.0.0 或具体的外部 IP。
