# RAG功能使用指南

本文档介绍如何使用netOpsAgent的RAG（检索增强生成）功能。

---

## 功能概述

RAG功能允许您上传私有知识文档（TXT格式），系统会自动将文档内容向量化并存储到知识库中。在进行通用问答时，系统会自动检索相关知识并用于增强AI的回答。

### 技术架构
- **向量数据库**: ChromaDB（轻量级，无需额外服务）
- **Embedding模型**: bge-small-zh-v1.5（本地运行，中文优化）
- **文档格式**: TXT文本文件
- **文本分割**: 500字符/块，100字符重叠

---

## 使用步骤

### 1. 上传知识文档

1. 访问知识库管理页面：http://127.0.0.1:8000/static/knowledge.html
2. 准备TXT格式的知识文档
3. 通过以下方式上传：
   - **拖拽上传**: 将文件拖拽到上传区域
   - **点击上传**: 点击"选择文件上传"按钮选择文件
4. 等待处理完成，系统会显示文档已分割为多少个知识块

### 2. 查看知识库

在知识库管理页面可以看到：
- **统计信息**: 文档数量、知识块数量
- **文档列表**: 已上传的文档，包括文件名、块数量、上传时间
- **删除操作**: 点击删除按钮可以移除不需要的文档

### 3. 使用RAG增强聊天

1. 返回主界面：http://127.0.0.1:8000/
2. 勾选"启用知识库检索"选项（默认已勾选）
3. 在输入框中提问
4. 系统会自动：
   - 🔍 检索知识库中的相关内容
   - 📚 显示找到的知识来源
   - 💬 基于检索到的知识生成回答

## 知识文档编写建议

### 文档结构

建议使用清晰的层次结构：

```
# 主题标题

## 子主题1

### 详细内容1
具体说明...

### 详细内容2
具体说明...

## 子主题2

### 详细内容3
具体说明...
```

### 内容组织

1. **分段明确**: 使用空行分隔不同段落
2. **标题清晰**: 使用标题标记重要概念
3. **列表结构**: 使用列表组织步骤或要点
4. **关键词丰富**: 包含用户可能搜索的关键词

## API接口说明

### 上传文档
```http
POST /api/v1/knowledge/upload
Content-Type: multipart/form-data

file: <TXT文件>
```

**响应**:
```json
{
  "status": "success",
  "doc_id": "doc_20260204_abc123",
  "filename": "network_guide.txt",
  "chunks": 15,
  "message": "文档上传成功，已分割为15个知识块"
}
```

### 获取文档列表
```http
GET /api/v1/knowledge/list
```

**响应**:
```json
{
  "status": "success",
  "documents": [
    {
      "doc_id": "doc_20260204_abc123",
      "filename": "network_guide.txt",
      "chunk_count": 15,
      "upload_time": "2026-02-04T10:00:00"
    }
  ],
  "total": 1
}
```

### 删除文档
```http
DELETE /api/v1/knowledge/{doc_id}
```

**响应**:
```json
{
  "status": "success",
  "doc_id": "doc_20260204_abc123",
  "deleted_chunks": 15,
  "message": "文档已删除"
}
```

### 获取统计信息
```http
GET /api/v1/knowledge/stats
```

**响应**:
```json
{
  "status": "success",
  "stats": {
    "total_documents": 5,
    "total_chunks": 120,
    "has_knowledge": true
  }
}
```

### RAG增强聊天
```http
POST /api/v1/chat/general/stream
Content-Type: application/json

{
  "message": "如何排查端口不通问题？",
  "use_rag": true
}
```

**响应**: Server-Sent Events (SSE) 流式推送

## 常见问题

### Q: 支持哪些文件格式？
A: 目前仅支持TXT文本文件。未来版本可能会支持PDF、Word等格式。

### Q: 文档大小有限制吗？
A: 建议单个文档不超过10MB。过大的文档会影响处理速度。

### Q: 如何提高检索准确性？
A: 
- 使用清晰的标题和段落结构
- 包含丰富的关键词
- 避免过于口语化的表达
- 一个文档专注于一个主题

### Q: 知识库数据存储在哪里？
A: 
- 向量数据库：`runtime/knowledge_base/vectordb/`
- 原始文件：`runtime/knowledge_base/documents/`

### Q: 如何备份知识库？
A: 直接复制 `runtime/knowledge_base/` 整个目录即可。

### Q: 重启服务器后知识库会丢失吗？
A: 不会。知识库数据是持久化存储的，重启后会自动加载。

### Q: 可以关闭RAG功能吗？
A: 可以。在主界面取消勾选"启用知识库检索"即可。

## 性能优化建议

### 文档数量
- 建议知识库文档数量控制在100个以内
- 单个文档建议不超过5000字

### 检索性能
- Top-K设置为5-10可以获得较好的性能和效果平衡
- 相关性阈值设置为0.5可以过滤掉不太相关的结果

### 模型加载
- 首次使用时会下载并加载Embedding模型（约95MB）
- 后续使用会直接加载本地缓存的模型
- 模型仅在首次调用RAG功能时加载，不影响启动速度

## 故障排查

### 上传失败
1. 检查文件格式是否为TXT
2. 检查文件编码是否为UTF-8
3. 查看服务器日志获取详细错误信息

### 检索无结果
1. 确认知识库中有相关文档
2. 尝试使用更具体的关键词
3. 检查文档内容是否包含相关信息

### 模型加载失败
1. 检查网络连接（首次需要下载模型）
2. 检查磁盘空间是否充足
3. 查看日志中的错误信息

## 技术细节

### 文本分割策略
采用递归字符分割策略：
1. 优先按段落分割（`\n\n`）
2. 其次按句子分割（中文标点：。！？）
3. 最后按字符数分割
4. 自动合并过小的文本块（小于50字符）

### 向量检索
- 使用余弦相似度计算相关性
- 相关性分数范围：0-1（越大越相关）
- 默认阈值：0.5（可在代码中调整）

### Prompt增强
检索到的知识会被整合到系统提示词中：
```
【参考资料开始】
[来源 1: network_guide.txt，相关度: 0.85]
<文本内容>

[来源 2: troubleshooting.txt，相关度: 0.72]
<文本内容>
【参考资料结束】
```

## 证据来源展示功能

### 功能概述

证据来源展示功能让您在使用RAG增强聊天时，能够清晰地看到AI回答所依据的知识来源，并支持点击查看完整的源文档内容。

### 主要特性

1. **证据来源面板**: 在AI回答下方显示所有相关文档
2. **相关度评分**: 每个证据来源都有相关度评分（0-100%）
3. **预览文本**: 显示文档的前200字符预览
4. **完整文档查看**: 点击证据卡片可查看完整文档内容
5. **颜色标识**: 根据相关度使用不同颜色标识
   - 高相关度（>70%）: 高亮颜色
   - 中等相关度（40-70%）: 中性颜色
   - 低相关度（<40%）: 低亮度颜色

### 使用方法

#### 1. 查看证据来源

当您提问后，如果系统从知识库检索到相关文档，会在AI回答下方显示证据来源面板：

```
📚 证据来源 (3)
┌─────────────────────────────────────────┐
│ 📄 访问关系开通流程.txt          [85%] │
│ 访问关系开通需要提交工单，包括源IP...  │
└─────────────────────────────────────────┘
┌─────────────────────────────────────────┐
│ 📄 网络权限申请指南.txt          [72%] │
│ 网络权限申请流程包括填写申请表...      │
└─────────────────────────────────────────┘
┌─────────────────────────────────────────┐
│ 📄 防火墙配置说明.txt            [65%] │
│ 防火墙配置需要在目标服务器上...        │
└─────────────────────────────────────────┘
```

#### 2. 查看完整文档

点击任意证据卡片，会弹出模态框显示完整文档内容：

- **文档标题**: 显示文件名
- **完整内容**: 显示文档的完整文本
- **滚动查看**: 支持滚动查看长文档
- **关闭按钮**: 点击关闭按钮或模态框外部区域关闭

#### 3. 历史消息中的证据来源

历史消息中的证据来源也会被保存和显示，您可以随时回顾之前对话中的知识来源。

#### 文档查询API

```http
GET /api/v1/knowledge/document/{doc_id}
```

返回完整文档内容，支持：
- 文档ID验证（防止路径遍历攻击）
- 文档大小限制（最大10MB）
- LRU缓存（最多缓存100个文档）
- 速率限制（每分钟10次）

### 配置选项

#### 启用/禁用证据来源功能

在 `.env` 文件中配置：

```bash
# RAG Evidence Sources Feature Flag
ENABLE_EVIDENCE_SOURCES=true  # 启用证据来源功能
```

设置为 `false` 可以禁用证据来源功能，系统将不发送 `evidence_sources` 事件。

#### 证据来源数量限制

系统默认最多返回5个证据来源，按相关度从高到低排序。

#### 相关度阈值

系统会自动过滤相关度低于0.05的文档，确保返回的证据来源都是相关的。

### 性能优化

#### 缓存策略

- 使用LRU缓存策略缓存文档内容
- 最多缓存100个文档
- 缓存命中时查询耗时可降低90%以上

#### 懒加载

- 证据来源面板仅显示预览文本（200字符）
- 完整文档内容仅在用户点击时才加载
- 减少初始数据传输量

#### 压缩传输

- 对超过1MB的文档内容进行gzip压缩
- 减少网络传输时间

### 安全防护

#### 路径遍历防护

系统会验证文档ID的合法性，防止路径遍历攻击（如 `../`、`..\\`）。

#### XSS防护

前端对文档内容进行HTML转义，防止XSS攻击。

#### 速率限制

每个客户端IP每分钟最多查询10次文档，防止恶意爬取。

系统会自动记录以下监控信息：

### 常见问题

#### Q: 为什么有些消息没有证据来源？

A: 以下情况不会显示证据来源：
- 问题不是知识性问题（如访问关系数据查询）
- 知识库中没有相关文档
- 相关度评分都低于阈值（0.05）
- 证据来源功能被禁用（ENABLE_EVIDENCE_SOURCES=false）

#### Q: 证据来源的相关度评分是如何计算的？

A: 相关度评分基于向量相似度计算：
- 使用余弦相似度算法
- 分数范围：0-1（0表示完全不相关，1表示完全相关）
- 显示时转换为百分比（0-100%）

#### Q: 为什么点击证据卡片后显示"文档不存在"？

A: 可能的原因：
- 文档已被删除
- 文档ID无效
- 向量数据库和文档存储不同步

解决方法：
- 重新上传文档
- 检查知识库完整性
- 查看服务器日志获取详细错误信息

#### Q: 如何提高证据来源的准确性？

A: 建议：
- 使用清晰的文档标题和结构
- 包含丰富的关键词
- 避免过于口语化的表达
- 一个文档专注于一个主题
- 定期更新和维护知识库

#### Q: 证据来源会影响聊天性能吗？

A: 影响很小：
- 证据来源数据在RAG检索时已经获取，无额外开销
- 预览文本限制为200字符，数据量小
- 完整文档采用懒加载，仅在点击时加载
- 使用LRU缓存提高查询速度

#### Q: 历史消息中的证据来源会保存吗？

A: 会保存：
- 证据来源信息会随消息一起保存到localStorage
- 刷新页面或重新打开会话时，证据来源会正确显示
- 证据卡片仍然可以点击查看完整文档

### 故障排查

#### 证据来源不显示

1. 检查 `.env` 文件中 `ENABLE_EVIDENCE_SOURCES` 是否为 `true`
2. 确认知识库中有相关文档
3. 检查浏览器控制台是否有JavaScript错误
4. 查看服务器日志确认是否发送了 `evidence_sources` 事件

#### 文档预览加载失败

1. 检查网络连接
2. 确认文档ID有效
3. 检查是否超过速率限制（每分钟10次）
4. 查看服务器日志获取详细错误信息

#### 相关度评分异常

1. 检查文档内容质量
2. 确认Embedding模型正常加载
3. 尝试重新上传文档
4. 查看RAG检索日志
