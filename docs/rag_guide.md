# RAG功能使用指南

本文档介绍如何使用netOpsAgent的RAG（检索增强生成）功能。

---

## 功能概述

RAG功能允许您上传私有知识文档（TXT格式），系统会自动将文档内容向量化并存储到知识库中。在进行通用问答时，系统会自动检索相关知识并用于增强AI的回答。

### 技术架构
- **向量数据库**: ChromaDB（轻量级，无需额外服务）
- **Embedding模型**: bge-small-zh-v1.5（本地运行，中文优化）
- **文档格式**: TXT文本文件
- **文本分割**: 500字符/块，100字符重叠

---

## 使用步骤

### 1. 上传知识文档

1. 访问知识库管理页面：http://127.0.0.1:8000/static/knowledge.html
2. 准备TXT格式的知识文档
3. 通过以下方式上传：
   - **拖拽上传**: 将文件拖拽到上传区域
   - **点击上传**: 点击"选择文件上传"按钮选择文件
4. 等待处理完成，系统会显示文档已分割为多少个知识块

### 2. 查看知识库

在知识库管理页面可以看到：
- **统计信息**: 文档数量、知识块数量
- **文档列表**: 已上传的文档，包括文件名、块数量、上传时间
- **删除操作**: 点击删除按钮可以移除不需要的文档

### 3. 使用RAG增强聊天

1. 返回主界面：http://127.0.0.1:8000/
2. 勾选"启用知识库检索"选项（默认已勾选）
3. 在输入框中提问
4. 系统会自动：
   - 🔍 检索知识库中的相关内容
   - 📚 显示找到的知识来源
   - 💬 基于检索到的知识生成回答

## 知识文档编写建议

### 文档结构

建议使用清晰的层次结构：

```
# 主题标题

## 子主题1

### 详细内容1
具体说明...

### 详细内容2
具体说明...

## 子主题2

### 详细内容3
具体说明...
```

### 内容组织

1. **分段明确**: 使用空行分隔不同段落
2. **标题清晰**: 使用标题标记重要概念
3. **列表结构**: 使用列表组织步骤或要点
4. **关键词丰富**: 包含用户可能搜索的关键词

## API接口说明

### 上传文档
```http
POST /api/v1/knowledge/upload
Content-Type: multipart/form-data

file: <TXT文件>
```

**响应**:
```json
{
  "status": "success",
  "doc_id": "doc_20260204_abc123",
  "filename": "network_guide.txt",
  "chunks": 15,
  "message": "文档上传成功，已分割为15个知识块"
}
```

### 获取文档列表
```http
GET /api/v1/knowledge/list
```

**响应**:
```json
{
  "status": "success",
  "documents": [
    {
      "doc_id": "doc_20260204_abc123",
      "filename": "network_guide.txt",
      "chunk_count": 15,
      "upload_time": "2026-02-04T10:00:00"
    }
  ],
  "total": 1
}
```

### 删除文档
```http
DELETE /api/v1/knowledge/{doc_id}
```

**响应**:
```json
{
  "status": "success",
  "doc_id": "doc_20260204_abc123",
  "deleted_chunks": 15,
  "message": "文档已删除"
}
```

### 获取统计信息
```http
GET /api/v1/knowledge/stats
```

**响应**:
```json
{
  "status": "success",
  "stats": {
    "total_documents": 5,
    "total_chunks": 120,
    "has_knowledge": true
  }
}
```

### RAG增强聊天
```http
POST /api/v1/chat/general/stream
Content-Type: application/json

{
  "message": "如何排查端口不通问题？",
  "use_rag": true
}
```

**响应**: Server-Sent Events (SSE) 流式推送

## 常见问题

### Q: 支持哪些文件格式？
A: 目前仅支持TXT文本文件。未来版本可能会支持PDF、Word等格式。

### Q: 文档大小有限制吗？
A: 建议单个文档不超过10MB。过大的文档会影响处理速度。

### Q: 如何提高检索准确性？
A: 
- 使用清晰的标题和段落结构
- 包含丰富的关键词
- 避免过于口语化的表达
- 一个文档专注于一个主题

### Q: 知识库数据存储在哪里？
A: 
- 向量数据库：`runtime/knowledge_base/vectordb/`
- 原始文件：`runtime/knowledge_base/documents/`

### Q: 如何备份知识库？
A: 直接复制 `runtime/knowledge_base/` 整个目录即可。

### Q: 重启服务器后知识库会丢失吗？
A: 不会。知识库数据是持久化存储的，重启后会自动加载。

### Q: 可以关闭RAG功能吗？
A: 可以。在主界面取消勾选"启用知识库检索"即可。

## 性能优化建议

### 文档数量
- 建议知识库文档数量控制在100个以内
- 单个文档建议不超过5000字

### 检索性能
- Top-K设置为5-10可以获得较好的性能和效果平衡
- 相关性阈值设置为0.5可以过滤掉不太相关的结果

### 模型加载
- 首次使用时会下载并加载Embedding模型（约95MB）
- 后续使用会直接加载本地缓存的模型
- 模型仅在首次调用RAG功能时加载，不影响启动速度

## 故障排查

### 上传失败
1. 检查文件格式是否为TXT
2. 检查文件编码是否为UTF-8
3. 查看服务器日志获取详细错误信息

### 检索无结果
1. 确认知识库中有相关文档
2. 尝试使用更具体的关键词
3. 检查文档内容是否包含相关信息

### 模型加载失败
1. 检查网络连接（首次需要下载模型）
2. 检查磁盘空间是否充足
3. 查看日志中的错误信息

## 技术细节

### 文本分割策略
采用递归字符分割策略：
1. 优先按段落分割（`\n\n`）
2. 其次按句子分割（中文标点：。！？）
3. 最后按字符数分割
4. 自动合并过小的文本块（小于50字符）

### 向量检索
- 使用余弦相似度计算相关性
- 相关性分数范围：0-1（越大越相关）
- 默认阈值：0.5（可在代码中调整）

### Prompt增强
检索到的知识会被整合到系统提示词中：
```
【参考资料开始】
[来源 1: network_guide.txt，相关度: 0.85]
<文本内容>

[来源 2: troubleshooting.txt，相关度: 0.72]
<文本内容>
【参考资料结束】
```
